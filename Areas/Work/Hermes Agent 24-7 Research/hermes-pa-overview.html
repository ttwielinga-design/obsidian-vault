  • Bot token from Discord Developer Portal (DISCORD_BOT_TOKEN)
  • User/role allowlist (DISCORD_ALLOWED_USERS or DISCORD_ALLOWED_ROLES)
  • Message Content Intent and Server Members Intent must be enabled in Developer Portal (critical — #1 cause of silent failures)
  • discord.py library
  • libopus for voice channel support (bundled on Windows)

WhatsApp (Available, Two Adapter Options)

● Available but not configured — whatsapp: {} in config (empty block)

Option A: Baileys Bridge (whatsapp adapter)

  • Backend: Node.js subprocess running Baileys (unofficial WhatsApp Web protocol)
  • Setup: QR code scan via hermes whatsapp wizard
  • No public URL needed — polls via local HTTP bridge
  • Supports: Text, images, files, voice notes, streaming, group chats
  • Limitations: Ban risk (unofficial API), no typing indicator, no read receipts, no interactive buttons, no reactions, no threads
  • Session: ~/.hermes/platforms/whatsapp/session (encryption keys)

Option B: Business Cloud API (whatsapp_cloud adapter)

  • Backend: Official Meta WhatsApp Business Cloud API (Graph API v20.0)
  • Setup: Meta app + WABA + System User token, public HTTPS URL (cloudflared/ngrok)
  • Supports: Text, images, voice notes (ogg/opus via ffmpeg), documents, video, native interactive buttons, read receipts, typing indicator, streaming, HMAC verification
  • Limitations: DMs only (no groups), 24-hour conversation window, max 5 numbers in dev mode, rate limit 80 msg/sec, requires ffmpeg on PATH for voice bubbles
  • Setup complexity: High — 8 steps including Meta Business account, webhook config, tunnel

Recommendation

Baileys bridge for quick start (personal use, QR scan, group support). Switch to Cloud API if ban risk is unacceptable or if you need interactive buttons for approval flows.
