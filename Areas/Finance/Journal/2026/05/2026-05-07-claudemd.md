## High-Risk Invariants

### Classification pipeline

Transaction categorization runs in this exact order:

`OWN_IBANS -> memory.ibans -> memory.names -> savedRules -> CAT_RULES -> Uncategorized`

- `savedRules` live in `pf_rules`.
- Each saved rule is `{ matchType, matchValue, category }`.
- Valid `matchType` values:
  - `counterparty_name`
  - `counterparty_iban`
  - `description_prefix`
- `parseCSV()` precomputes `tx.descPrefix` using `normalizeDescPrefix()`.
- When working with parsed transactions, prefer `tx.descPrefix ?? normalizeDescPrefix(tx.desc)`.

### Single categorized source of truth

- All categorized transactions should flow through `allCategorized`.
- Do not add fresh `classify()` loops elsewhere for month, YTD, prior-year, or chart logic.
- New derived metrics should read from `allCategorized.get(key)`.

### Cash flow semantics

- `computeCashFlow()` ignores `Transfer`.
- Negative amounts in expense categories add to expense totals.
- Positive amounts in expense categories reduce that expense bucket. They are refunds/credits, not income.
- `wealthRetentionRate` is distinct from `savingsRate`: it adds back `invested` instead of redefining `surplus`.

### Net worth semantics

- `latestCashBal` always means the end balance of the most recently stored month.
- `netWorth` always uses `latestCashBal`, not the selected month.
- Property and mortgage enter net worth only at or after `settings.purchaseMonth`.
- Treat `settings.mortgageOpeningBalance` + amortization as the long-term mortgage source of truth. `mortgageOutstanding` is derived current state / compatibility data, not the opening principal.
- If you need the selected month's closing balance, use:

```js
const selectedCashBal = categorized[categorized.length - 1]?.balance ?? 0;
```

### Year semantics

- `currentYear` is the real current calendar year and is used for global YTD stats.
- `selectedYear` comes from `selectedMonth` and is used for month-specific YTD and YoY views.
- Do not conflate them when adding comparisons or summaries.

### Declaration-order pitfalls

- `src/Dashboard.jsx` is order-sensitive.
- A `useEffect` dependency array cannot reference a `const`, `useMemo`, or `useCallback` declared later in the file.
- **Critical**: `useMemo`/`useCallback` dependency arrays are evaluated eagerly at call time, not lazily. Referencing a `useCallback` that appears later in the file causes a TDZ `ReferenceError` — visible as a blank white page in the production build. Always place a `useMemo` or `useCallback` **after** every function it names in its dependency array.
- Plain derived constants also must appear before memos that depend on them.
- When in doubt, move the dependent code lower or compute the value inside the effect.

### History state

- `networthHistory` is React state mirrored to `pf_networth_history`.
- Inside the component, use `networthHistory`, not `lsGet('pf_networth_history', ...)`.
- Restore/migration may recompute `networthHistory` when stored history is stale or on an older derived-history version.

## Storage Contract

Primary localStorage keys:

| Key | Purpose |
| --- | --- |
| `pf_months` | Source of truth for uploaded transactions, keyed by `YYYY-MM` |
| `pf_rules` | Saved auto-categorization rules |
| `pf_memory` | Learned IBAN and counterparty-name mappings |
| `pf_overrides` | Manual per-transaction category overrides |
| `pf_settings` | User settings and budgets |
| `pf_portfolio` | Portfolio holdings |
| `pf_networth_history` | Historical monthly net worth and cash flow snapshots |
| `pf_csvName` | Last uploaded CSV filename |
| `pf_view_mode` | Cash flow mode: `monthly`, `ytd`, or `yoy` |
| `pf_txs` | Legacy backward-compat alias for the selected month's raw transactions |
| `pf_lots` | Map of `holdingId → lot[]` for lot-level investment tracking. Lots: `{ id, date, qty, pricePerShare, amountEUR, source, txId? }`. Holdings with no lots fall back to legacy `{qty, avgPrice}` via `effectiveHolding()` in `src/lib/lots.js`. |

`pf_settings.autoInvest.brokerIbans` is an array of IBANs the user wires investment money to. Outgoing transactions to these IBANs are summed in the `brokerCashBalance` derived memo: `sum(transfers to broker IBANs) − sum(lot.amountEUR) + brokerCashAdjustment`. Broker lots dated before the first tracked broker transfer are treated as opening holdings and excluded from the subtraction.

`pf_settings.brokerCashAdjustment` (number, default `0`) is a manual offset added to `brokerCashBalance` to correct for untracked deposits or rounding.

Portfolio holdings may carry optional price-service fields: `priceUpdatedAt` (ISO string), `currentPriceCurrency` (string), `priceSymbol` (string — Yahoo Finance symbol e.g. `VWCE.DE`). These are undefined when absent and must be read defensively. `normalizePortfolioHolding` in `src/lib/defaults.js` preserves them if present.

`holding.targetWeight` is stored as a decimal and feeds the portfolio weighted TER calculation.

`pf_networth_history` entries store monthly snapshots with `month`, `netWorth`, `cash`, `property`, `investments`, `mortgage`, `income`, `surplus`, `savingsRate`, `invested`, `wealthRetentionRate`, `exp`, and history-version metadata.

If you change a stored shape, update all of these as needed:

- Default values (`DEF_*`)
- Persistence effects
- Backup export
- Restore/import logic
- Any migration logic

## UI Guardrails

- Theme tokens are centralized in `T`; prefer them over hardcoded inline colors.
- Icon set is capped at **25 icons** (`alert`, `cashflow`, `check`, `chevDown`, `chevLeft`, `chevRight`, `close`, `command`, `copy`, `download`, `equity`, `eye`, `filter`, `goals`, `help`, `moreH`, `overview`, `portfolio`, `queue`, `search`, `settings`, `sparkle`, `subscriptions`, `transactions`, `upload`). No new icons without explicit user approval — if you need a glyph that isn't there, stop and ask before adding it.
- Reuse `card(extra)` for card containers when possible.
- Top-level content is section-driven via `activeSection`:
  - `overview`
  - `cashflow`
  - `portfolio` — IA: Summary header (Investment Portfolio | Cash at Broker | Portfolio Total) → Holdings reference → Action row (Add / Refresh prices / Import broker trades)
  - `transactions`
  - `goals` (new; read-only progress bars against targets in `settings`)
  - `subscriptions` (new; recurring-charge detection via `detectSubscriptions()` in `src/lib/subscriptions.js`)
- `visibleTxs` intentionally caps the rendered transaction table at 100 rows.
- Design tokens live in `src/styles/tokens.css` (CSS custom properties). The `T` object in `src/Dashboard.jsx` mirrors the same palette so inline styles read the same values. Prefer `var(--…)` in new markup; update `T` only if a token changes.
- Command palette (`Cmd/Ctrl+K`) and the `g-x` keyboard chords are wired via `useCommandPalette()` and `useKeyboardNav()` in `src/hooks/`. The `?` key opens the shortcuts overlay.

## Data & Privacy

- No `fetch()` or `XMLHttpRequest` for user data (transactions, IBANs, accounts, goals, settings). All user data is parsed client-side from CSV and persisted only to localStorage / IndexedDB — never sent to a server.
- Market-data fetches (FX rates, security prices) are permitted in `src/lib/` and **only** `src/lib/`. They must:
  - Send no user-identifiable data
  - Fail gracefully (the app must remain usable offline)
  - Live behind a service module, never inline in components

## Backup Format

`handleExportData()` (via `buildBackupPayload` in `src/lib/backup.js`) currently writes:

```js
{
  version: 9,
  historyVersion: NETWORTH_HISTORY_VERSION,
  exportedAt: 'ISO string',
  months: {},
  settings: {},  // includes brokerCashAdjustment and autoInvest.brokerIbans
  portfolio: [],
  rules: [],
  memory: {},
  overrides: {},
  networthHistory: [],
  csvName: '',
  viewMode: 'monthly',
  lots: {},
}
```

Restore (`restoreBackupState`) handles v1 through v9. Migration expectations:

- v1 recomputes history when needed.
- v2–v4 default missing fields progressively.
- v6–v7 handled via `mergeDefaultSettings` (brokerCashAdjustment defaults to 0).
- v8 removes `brokerCash` and `lotLinks`; `brokerCashAdjustment` lives in settings.
- v9 is the current backup schema. It migrates legacy portfolio price symbols to `priceSymbol` and no longer stores `symbolMap`.

Restore expects at least `version` and `months`. Reads from in-memory state passed as argument — not from `localStorage` directly.
